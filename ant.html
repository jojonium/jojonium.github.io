How does it work?

This page is a customizable implementation of Langton's Ant, a cellular automaton invented by Chris Langton in 1986. Langton's Ant is interesting because its very simple set of rules can lead to very complex emergent behavior. The simulation deals with computerized "ants" whose movements are controlled by a set of rules decided by the user. In its most basic form the simulation creates a single ant, which follows these rules:

  • At a white square, turn 90° right, change the color of the square to black, then move forward one square
  • At a black square, turn 90° left, change the color of the square to white, then move forward one square

You can try this basic configuration now by pressing "Go!"

At first, the ant's movement appears to be essentially random, retracing its steps many times and turning in unpredictable ways. However, after about 10,000 iterations the ant begins moving in a pattern. It constructs a repeating "highway" in one direction, eventually reaching the edge of the grid. This is the really fascinating aspect of Langton's Ant: order arising out of chaos.

Extensions

The basic implementation of Langton's Ant is cool, but it does the same thing every time, which gets boring. Several ways to extend the functionality of the ant have been suggested over the years. For example, multiple ants can be simulated by pressing the "Add Ant" button, each with a unique starting position and direction. This can show how multiple ants interact with each other and give rise to different ordered patterns.

Another way to extend Langton's Ant is to add more colors, and more rules. By clicking the "Add Rule" button you can define any number of rules for the ants to obey. In addition to the basic "turn right" (R) and "turn left" (L), ants can also continue straight ahead (C) or turn 180° around (U).

The combination of these extensions can lead to some very interesting and unpredictable outcomes. Sometimes ordered patterns arise out of chaos, and sometimes symmetry and repetition degenerates into randomness. Unpredictable patterns often emerge from these simple rules.

The point of this page is to make it easy to experiment and play with different combinations of rules and ants. The next section explains how to use the provided options to do just that.

How to use this page

The first three options (Grid Size, Pixel Size, and Interval) are global variables that affect how the simulation is displayed. Grid size is the size of the length and width of the simulation board (it's square, so one input sets both). Pixel size is the size of the simulation squares in pixels. The Interval is the amount of time between each iteration, in milliseconds. Any invalid inputs in these fields will be replaced by the default values.

To add more ants, click the "Add Ant" button. Each ant has three attributes you can set: its starting X position, measued in number of squares to the right of the left side of the board; starting Y position, measured in number of squares below the top of the board, and starting direction. Direction is a number from 0 to 3, representing which the ant is facing when the simulation begins:

  • 0: facing right
  • 1: facing up
  • 2: facing left
  • 3: facing right

Any invalid input in these fields will be replaced by a random value within the confines of the simulation board.

The box on the right allows you to define any number of custom rules. Each entry in the rules box has a color and a letter, representing what an ant does on a square of that color. There are four valid rule letters:

  • R: turn 90° Right
  • L: turn 90° Left
  • C: don't turn (Continue straight)
  • U: turn 180° (make a U-turn)

You can also customize the colors by clicking on the colored box to the left of the rule letter.

At the start of each iteration the ant will follow the rule for the color of the square it's on, the cycle the color of the square it's on to the next color in the list of rules, then move one unit in whichever direction it's facing. Any invalid input will be replaced by a random valid rule.

If "Wrap Around" is checked, when ants reach one of the borders of the board they wrap around to the other side. If this option is turned off ants simply stop moving once they reach the edge of the board. The "Clear" button clears all input options and reloads the page. The "Pause" button can be used to pause the simulation and observe a single iteration.

Finally, the "Go!" button starts the simulation. Please note that none of the options can be changed after the simulation has started.
